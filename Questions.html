Html Questions and Answers.

1. What is HTML, and why is it important in web development and
explain the purpose of HTML and its role in structuring web content.?

Answer: HTML(hypertext markup language) is the standard language to structure web page content.
It's the skeleton of a web page. It is important in a web page because it organises content into elements like headings,
paragraphs, links, images, and more, making web pages readable and accessible.
Without HTML, web pages would be just plain text without any structure or meaning. Browers read HTML to display the content correctly.

2. What is the difference between an id and a class in HTML
Describe the usage and differences between id and class attributes.?

Answer: "id" identifies/targets one specific element.
There can't be more than one element with the same id, while "class" can apply to multiple elements.

3. What are semantic HTML elements, and why are they important?
Define semantic elements and explain how they contribute to accessibility and SEO.

Answer: Semantic elements clearly describe their meaning to the developer and browser. e.g.: footer, section.
They make code more organized and improve accessibility.
Accessibility: Semantic tags help screen readers and assistive technologies understand the page structure.
This improves navigation for users with disabilities.
SEO: Search engines use semantic tags to understand the content better, which can improve search rankings.

4. Explain the concept of the DOM (Document Object Model).
What is the DOM, and how does it relate to the HTML structure of a web page?

Answer: It is like a tree-like structure representing a webpage's HTML elements as objects.
JavaScript can access, change, add, or delete parts of the page through the DOM.
The HTML document you write is converted by the browser into the DOM tree.
This allows JavaScript to:
Access elements (e.g., getElementById)
Modify content (e.g., change text or images)
Add or remove elements dynamically
React to events (like clicks or form submissions)

7. What is the difference between inline, block, and inline-block elements in HTML?

Answer: Inline elements do not start on a new line and only take up as much width as necessary.

CSS

9. What is the box model in CSS, and what are its components?

Answer: CSS box model is a box that wraps around every HTML element.
The box model describes the rectangular boxes generated for elements in the document tree and consists of:
1. Content: The actual content of the box, where text and images appear.
2. Padding: The space between the content and the border, inside the element.
3. Border: A line surrounding the padding and content.
4. Margin: The space outside the border, separating the element from others.

10. Explain the difference between padding and margin in CSS.

Answer: Padding is the space between the content and the border of an element,
while margin is the space outside the border, separating the element from other elements.
Padding is inside the element, while margin is outside.

11. What are CSS selectors, and how do they work?
Describe different types of selectors (e.g., element, class, id, attribute) and how they are used to target HTML elements.

Answer: CSS selectors are patterns used to select and style HTML elements.
Different types of selectors include:
1. Element selector: Selects elements by their tag name (e.g., p, h1).
2. Class selector: Selects elements with a specific class attribute (e.g., .classname).
3. ID selector: Selects a single element with a specific id attribute (e.g., #idname).
4. Attribute selector: Selects elements based on the presence or value of an attribute (e.g., [type="text"]).

12. What is Flexbox, and how does it work in CSS?
Explain Flexbox layout and its properties (e.g., justify-content, align-items, flex-direction) with examples.

Answer: Flexbox is a CSS layout model that makes it easy to arrange items in a row or column, evenly space them, and align them nicely.
Key properties include:
justify-content: aligns items horizontally.
align-items: aligns items vertically.
flex-direction: sets the main axis (row or column).

13. What is the difference between position: relative and position: absolute in CSS?
Explain the positioning scheme in CSS and how these two properties behave.

Answer: position: relative positions an element relative to its normal position,
while position: absolute positions an element relative to its nearest positioned ancestor (not static).
Relative elements still take up space in the document flow, while absolute elements do not.

14. What is the purpose of media queries in CSS?
Explain how media queries are used for responsive design and provide an example.

Answer: Media queries are used to apply different styles based on the device's characteristics,
such as screen size or resolution. They help create responsive designs that adapt to various devices.
Example:
@media (max-width: 600px) {
body {
background-color: lightblue;
}
}
This changes the background color to light blue for screens smaller than 600px.

15. What is CSS Grid, and how does it differ from Flexbox?
Describe the CSS Grid layout system and compare it to Flexbox, explaining when to use each.

Answer: CSS Grid is a powerful two-dimensional layout system in CSS that allows you to design web layouts using rows and columns.
With Grid, you can place items precisely on a grid, control spacing, and create complex layouts easily.
You define a grid container and set the number of rows and columns.
Then you place items into specific grid cells or areas.
Flexbox is one-dimensional, focusing on arranging items in a single row or column.
Use CSS Grid when:
You need to align items in rows and columns
You’re building a full page layout
You want overlapping items (e.g., layers)
Use Flexbox when:
You need to align items in a single row or column
You want to distribute space between items
You’re building a simple navigation bar or list
You need to control the order of items

16. What are pseudo-classes and pseudo-elements in CSS?
Define pseudo-classes (e.g., :hover, :focus) and pseudo-elements (e.g., ::before, ::after) with examples of their usage.

Answer: Pseudo-classes are used to style elements based on their state or position,
while pseudo-elements are used to style specific parts of an element.
Examples:
Pseudo-class: a:hover { color: blue; } (changes link color on hover)
Pseudo-element: h1::before { content: "🔥 ";}
Pseudo-elements are often used to add decorative content without adding extra HTML elements.


JavaScript

17. What is the difference between var, let, and const in JavaScript?
Explain the scoping rules and differences between these three keywords for declaring variables.

Answer: "var" is function-scoped and can be redeclared,
"let" is block-scoped and cannot be redeclared in the same scope,
"const" is also block-scoped but cannot be reassigned or redeclared.
Use "let" for variables that may change, "const" for constants, and "var" for older code (not recommended).

18. What is the difference between null and undefined in JavaScript?
Describe the two types and explain their differences in JavaScript.

Answer: "null" is an intentional absence of value, while "undefined" means a variable has been declared but not assigned a value.
Null is an object type, while undefined is a type of its own.
Use null when you want to explicitly indicate no value, and use undefined when a variable has not been initialized.

19. What are JavaScript functions, and how do you define one?
Explain how functions are defined and invoked in JavaScript, and provide examples of both regular and arrow functions.

Answer: Functions are reusable blocks of code that perform a specific task.
You define a function using the "function" keyword, followed by its name and parameters.
Example of a regular function:
function add(a, b) {
return a + b;
}
Example of an arrow function:
const add = (a, b) => a + b;
You invoke a function by calling its name followed by parentheses.
Example: add(2, 3); // returns 5

20. What is the purpose of the "this" keyword in JavaScript?
Explain how this works in different contexts (e.g., in regular functions, arrow functions, and event handlers).

Answer: The "this" keyword refers to the context in which a function is called.
In regular functions, "this" refers to the global object (or undefined in strict mode).
In arrow functions, "this" retains the value of the enclosing lexical context.
In event handlers, "this" refers to the element that triggered the event.
Example:
const obj = {
name: "John",
greet: function() {
console.log("Hello, " + this.name);
}
};
obj.greet(); // Output: Hello, John
const arrowGreet = () => console.log("Hello, " + this.name);
